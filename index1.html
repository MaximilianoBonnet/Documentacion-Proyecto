    PLAN DE GESTIÓN DEL ALCANCE

    Este plan determina y especifica los mecanismos a implementar para definir el alcance del proyecto y para asegurar que el mismo esté bajo control en todo momento, haciendo que todos los cambios que se proponen sean evaluados y eventualmente aprobados por el equipo. La evaluación mencionada tiene especial relevancia porque incluye el impacto sobre el resto de los componentes de la configuración del proyecto, y no solamente el impacto del cambio en sí mismo. La autoridad para aceptar, rechazar o posponer un cambio propuesto por cualquiera de los stakeholders (partes involucradas) será analizada y acordada en el respectivo al equipo.

    1. Objetivo del Proyecto:

    Desarrollar una solución informática en línea de intercambio de productos o servicios Implementar una base de datos para el registro de usuarios, productos y transacciones. Documentar el desarrollo del sistema contemplando a posteriori el mantenimiento del mismo. Cumplir con el cronograma propuesto del proyecto y entrega del producto.

    Desarrollar una plataforma en línea, denominada DondeDeal, que utilice tecnología avanzada para proporcionar a los usuarios una herramienta eficiente para encontrar las mejores ofertas y precios de diversos productos en diferentes tiendas en línea.

    2. Funcionalidades Principales:

    2.1. Búsqueda de Productos:

    Implementar un sistema de búsqueda robusto que permita a los usuarios buscar productos específicos. Integrar un algoritmo de búsqueda inteligente que ofrezca resultados precisos y relevantes.

    2.2. Navegación por Categorías:

    Diseñar una interfaz intuitiva que permita a los usuarios navegar por categorías de productos. Desarrollar una estructura de categorías jerárquica para facilitar la búsqueda.

    2.3. Comparación de Precios:

    Obtener información en tiempo real de diversas tiendas en línea sobre precios y ofertas. Presentar una interfaz de comparación de precios que muestre las opciones disponibles para un producto específico.

    2.4. Perfiles de Usuario:

    Crear perfiles de usuario para personalizar la experiencia de búsqueda. Permitir a los usuarios guardar productos favoritos y recibir alertas de precios.

    2.5. Integración de Tiendas:

    Establecer conexiones con múltiples tiendas en línea para recopilar datos de productos y precios de manera efectiva. Garantizar una integración segura y eficiente mediante API o mecanismos similares.

    2.6. Filtrado Avanzado:

    Implementar opciones de filtrado avanzado para refinar las búsquedas según criterios específicos, como marcas, descuentos, etc.

    2.7. Experiencia del Usuario:

    Diseñar una interfaz de usuario atractiva y fácil de usar. Asegurar la accesibilidad y la capacidad de respuesta en diferentes dispositivos.

    3. Requisitos Técnicos:

    3.1. Tecnologías:

    Seleccionar tecnologías modernas para el desarrollo web y la gestión de bases de datos. Garantizar la escalabilidad del sistema para manejar un aumento en la carga de usuarios.

    3.2. Seguridad:

    Implementar medidas de seguridad robustas para proteger la información del usuario y los datos de la plataforma. Incorporar un sistema de autenticación seguro.

    3.3. Mantenimiento y Actualizaciones:

    Establecer un plan de mantenimiento regular para corregir errores y mejorar la funcionalidad. Planificar actualizaciones periódicas para agregar nuevas características y adaptarse a cambios en las API de las tiendas.

    4. Entregables:

    Prototipo funcional de la plataforma DondeDeal. Documentación detallada del código y manuales de usuario. Sistema desplegado y listo para su lanzamiento. Este alcance del proyecto sirve como guía inicial y puede ajustarse según los cambios y requisitos específicos que surjan durante el desarrollo.

    Planificación en Forma de Sprints utilizando Metodología Ágil

    Sprint 1: Definición y Configuración Inicial (Duración: 2 semanas) En este sprint inicial, nos centraremos en establecer los fundamentos del proyecto DondeDeal. Se llevarán a cabo las siguientes tareas:

    ● Definición detallada de los requisitos del usuario y los casos de uso.

    ● Configuración del entorno de desarrollo y elección de las tecnologías clave.

    ● Creación de prototipos básicos de las interfaces de usuario para validar la experiencia del usuario.

    Sprint 2: Desarrollo del Sistema de Búsqueda (Duración: 3 semanas) En este sprint, nos enfocaremos en implementar la funcionalidad de búsqueda, un componente crítico de DondeDeal. Las tareas específicas incluyen:

    ● Desarrollo del sistema de búsqueda, con algoritmos inteligentes para proporcionar resultados precisos.

    ● Implementación de la interfaz de usuario para la búsqueda de productos y navegación por categorías.

    ● Integración inicial con algunas tiendas en línea para pruebas de concepto.

    Sprint 3: Funcionalidades de Comparación de Precios y Filtrado (Duración: 3 semanas) Este sprint se centrará en mejorar la capacidad de comparación de precios y agregar funciones avanzadas de filtrado. Las actividades principales serán:

    ● Desarrollo de la interfaz de comparación de precios.

    ● Implementación de opciones de filtrado avanzado.

    ● Extensión de la integración con tiendas para garantizar datos precisos y actualizados.

    Sprint 4: Perfiles de Usuario y Personalización (Duración: 2 semanas) En este sprint, nos concentraremos en la creación de perfiles de usuario y funciones de personalización. Las tareas clave incluyen:

    ● Diseño e implementación de perfiles de usuario.

    ● Desarrollo de la funcionalidad de guardado de productos favoritos y alertas de precios.

    ● Pruebas de usabilidad para garantizar una experiencia fluida del usuario.

    Sprint 5: Optimización y Seguridad (Duración: 2 semanas) Durante este sprint, nos enfocaremos en optimizar el rendimiento del sistema y fortalecer la seguridad. Las actividades específicas serán:

    ● Optimización del rendimiento para garantizar tiempos de carga rápidos.

    ● Implementación de medidas de seguridad adicionales, como autenticación segura y encriptación de datos.

    ● Revisiones de código para garantizar buenas prácticas de desarrollo.

    Sprint 6: Pulido Final y Preparación para el Lanzamiento (Duración: 2 semanas) Este sprint final estará dedicado a realizar ajustes finales, correcciones de errores y preparación para el lanzamiento. Las tareas principales incluyen:

    ● Corrección de errores y refinamiento de la interfaz de usuario.

    ● Pruebas exhaustivas para identificar posibles problemas.

    ● Creación de documentación detallada para el código y manuales de usuario.

    ● Despliegue del sistema en un entorno de producción y preparación para el lanzamiento oficial de DondeDeal.

    Historia de Usuario

    Contexto:

    Se requiere crear un login de usuario, para eso precisamos disponibilizar el endpoint con los atributos de usuario. Crear un FE de login con el campo username y password. Con los botones para ingresar y otro recuperar contraseña.

    Requerimientos del Usuario (algunos)

    1. Registro de Usuario:
    • Los usuarios deben poder registrarse en el sitio.
    • Se requiere un proceso de verificación de correo electrónico.
    • Los usuarios deben tener la opción de recuperar su contraseña.
    2. Inicio de Sesión:
    • Los usuarios deben poder iniciar sesión de manera segura.
    • Integración con redes sociales para iniciar sesión.
    3. Exploración de Productos:
    • Los usuarios deben poder navegar y buscar productos fácilmente.
    • Funcionalidad de filtrado por categorías, precios, marcas, etc.
    • Búsqueda avanzada para encontrar productos específicos.
    4. Detalles del Producto:
    • Información detallada sobre cada producto.
    • Imágenes de alta calidad del producto.
    • Reseñas y calificaciones de productos.
    5. Carrito de Compras:
    • Capacidad para agregar y eliminar productos del carrito.
    • Mostrar el total del carrito y el subtotal.
    • Opción para aplicar códigos de descuento.
    6. Proceso de Pago:
    • Ofrecer múltiples opciones de pago (tarjeta de crédito, PayPal, transferencia bancaria, etc.).
    • Seguridad y encriptación de datos durante el proceso de pago.
    • Confirmación de pedido por correo electrónico después de completar el pago.
    7. Cuenta de Usuario:
    • Capacidad para gestionar la información de la cuenta (perfil, direcciones de envío, preferencias, etc.).
    • Historial de pedidos y seguimiento de envíos.
    8. Interfaz Responsiva:
    • Compatibilidad con dispositivos móviles y tabletas.
    • Experiencia de usuario consistente en diferentes tamaños de pantalla.
    9. Atención al Cliente:
    • Canal de comunicación para consultas y problemas (chat en vivo, formulario de contacto, etc.).
    • Base de conocimientos o sección de preguntas frecuentes.
    10. Personalización de la Experiencia:
    • Opciones de personalización de productos (colores, tallas, grabados, etc.).
    • Recomendaciones personalizadas según el historial de compras.
    Casos de Uso del Cliente:

    1. Registrarse:
    Descripción: Permite a un cliente nuevo crear una cuenta en el sistema proporcionando la información necesaria, como nombre, correo electrónico y contraseña.
    Flujo Principal:

    1. El cliente accede a la página de registro.
    2. Ingresa la información requerida.
    3. El sistema valida la información y crea la cuenta del cliente.

    2. Iniciar Sesión:
    Descripción: Permite a un cliente iniciar sesión en su cuenta para acceder a funciones personalizadas y realizar compras.
    Flujo Principal:

    1. El cliente ingresa su correo electrónico y contraseña.
    2. El sistema valida las credenciales y autentica al cliente.

    Casos de Uso del Administrador:

    1. Gestionar Productos:
    Descripción: Permite al administrador agregar, eliminar o modificar productos en el catálogo.
    Flujo Principal:

    1. El administrador accede a la sección de gestión de productos.
    2. Realiza las acciones necesarias (agregar, eliminar, modificar).

    2. Modificar Productos Existente:
    Descripción: Permite al administrador realizar cambios en la información de productos existentes.
    Flujo Principal:

    1. El administrador selecciona un producto existente
    2. Realiza las modificaciones necesarias.
    3. Guarda los cambios.

    Gestion de riesgos

    1. Gestión de Riesgos:

    ● En el contexto de la metodología ágil, la gestión de problemas es esencial para abordar cualquier obstáculo que pueda surgir durante el desarrollo del proyecto. Se seguirán los siguientes procedimientos:

    1.1. Identificación de Problemas:

    ● Establecer canales de comunicación abiertos para que los miembros del equipo informen de cualquier problema de inmediato.

    ● Utilizar herramientas de seguimiento de problemas para registrar y categorizar los problemas identificados.

    1.2. Reuniones de Retrospectiva del Sprint:

    ● Realizar reuniones de retrospectiva al final de cada sprint para evaluar el desempeño del equipo y discutir posibles problemas.

    ● Analizar las causas fundamentales de los problemas para abordarlos de manera efectiva.

    1.3. Priorización de Problemas:

    ● Clasificar los problemas según su impacto en el proyecto y su urgencia.

    ● Establecer prioridades para resolver problemas críticos que puedan afectar la entrega del producto.

    1.4. Asignación de Responsabilidades:

    ● Designar responsables claros para abordar problemas específicos.

    ● Fomentar la colaboración del equipo para encontrar soluciones y asignar tareas de resolución.

    2. Procedimientos de Gestión:

    2.1. Proceso de Desarrollo Ágil:

    ● Seguir el ciclo de vida ágil con sprints bien definidos, y realizar reuniones diarias de seguimiento para mantener a todos en el equipo actualizados sobre el progreso.

    2.2. Herramientas de Gestión de Proyectos:

    ● Utilizar herramientas de gestión de proyectos, como tableros Kanban o herramientas específicas de seguimiento de problemas, para visualizar el progreso y gestionar tareas.

    2.3. Actualización Continua de Documentación:

    ● Mantener documentación actualizada, incluyendo manuales de usuario, para reflejar cambios y mejoras en el sistema.

    ● Asegurarse de que todos los miembros del equipo tengan acceso a la documentación relevante.

    2.4. Pruebas y Control de Calidad:

    ● Integrar pruebas continuas en el desarrollo para identificar posibles problemas de calidad.

    ● Establecer procedimientos de control de calidad para garantizar que cada incremento cumpla con los estándares establecidos.

    2.5. Comunicación Efectiva:

    ● Facilitar una comunicación abierta y transparente dentro del equipo.

    ● Programar reuniones regulares para discutir avances, obstáculos y soluciones potenciales.

    2.6. Revisión de Hitos y Planificación Futura:

    ● Realizar revisiones periódicas de hitos alcanzados y evaluar el progreso hacia los objetivos del proyecto.

    ● Ajustar la planificación futura según las lecciones aprendidas y los cambios en los requisitos.

    Análisis de Riesgos para el Proyecto DondeDeal

    1.a) Integración con Tiendas en Línea:

    ● Riesgo: Posibles cambios en las API de las tiendas en línea podrían afectar la integración y la recopilación de datos.

    ● Estrategia de Mitigación: Establecer acuerdos de colaboración con las tiendas, realizar pruebas continuas de integración y contar con mecanismos de adaptación rápida a cambios en las API.

    1.b)Seguridad de Datos:

    ● Riesgo: Amenazas de seguridad, como ataques cibernéticos, podrían comprometer la información del usuario.

    ● Estrategia de Mitigación: Implementar medidas de seguridad robustas, como cifrado de datos, autenticación de usuarios y auditorías de seguridad regulares.

    1.c)Escalabilidad del Sistema:

    ● Riesgo: El aumento repentino en la carga de usuarios podría afectar el rendimiento y la escalabilidad del sistema.

    ● Estrategia de Mitigación: Realizar pruebas de carga para evaluar la capacidad del sistema, implementar estrategias de escalabilidad, como el uso de servicios en la nube, y tener un plan de contingencia para manejar picos inesperados.

    1.d)Cambios en Requisitos del Usuario:

    ● Riesgo: Cambios significativos en los requisitos del usuario podrían afectar el diseño y el desarrollo.

    ● Estrategia de Mitigación: Mantener una comunicación abierta con los stakeholders, realizar demostraciones frecuentes y adoptar prácticas ágiles para permitir adaptaciones continuas.

    1.e)Competencia en el Mercado:

    ● Riesgo: La competencia en el mercado podría influir en la adopción de la plataforma.

    ● Estrategia de Mitigación: Monitorear de cerca el mercado, mantenerse actualizado con las tendencias y adaptar la plataforma según las demandas cambiantes del usuario.

    1.f)Complejidad del Algoritmo de Búsqueda:

    ● Riesgo: La complejidad del algoritmo de búsqueda podría afectar el rendimiento y la experiencia del usuario.

    ● Estrategia de Mitigación: Realizar pruebas exhaustivas de rendimiento, optimizar el algoritmo según sea necesario y considerar soluciones de búsqueda en caché.

    1.g)Colaboración del Equipo:

    ● Riesgo: Problemas de colaboración y comunicación entre los miembros del equipo podrían afectar la productividad.

    ● Estrategia de Mitigación: Fomentar una cultura de equipo abierta, realizar reuniones regulares y utilizar herramientas de colaboración efectivas.

    1.h)Disponibilidad de Recursos:

    ● Riesgo: Limitaciones en recursos, como personal y financiamiento, podrían retrasar el desarrollo.

    ● Estrategia de Mitigación: Planificación cuidadosa de recursos, gestión eficiente del presupuesto y evaluación continua de las necesidades de personal.

    1.i)Rotación de Personal Clave:

    ● Riesgo: La salida de miembros clave del equipo podría afectar la continuidad y el conocimiento del proyecto.

    ● Estrategia de Mitigación: Documentar procesos clave, facilitar el conocimiento compartido y proporcionar incentivos para retener al personal clave.

    1.j)Capacitación y Curva de Aprendizaje:

    ● Riesgo: La curva de aprendizaje para nuevas tecnologías o procesos podría ralentizar el progreso.

    ● Estrategia de Mitigación: Planificar tiempo para la capacitación, fomentar un ambiente de aprendizaje continuo y asignar tareas de acuerdo con las fortalezas del equipo.

    1.k)Expectativas del Usuario:

    ● Riesgo: Divergencia entre las expectativas del usuario y la realidad del producto final.

    ● Estrategia de Mitigación: Involucrar a los usuarios de manera continua, obtener retroalimentación regular y realizar pruebas de usuario para validar y ajustar las expectativas.

    1.l)Comunicación Interna:

    ● Riesgo: Problemas de comunicación interna pueden causar malentendidos y errores.

    ● Estrategia de Mitigación: Establecer canales claros de comunicación, utilizar herramientas de colaboración efectivas y fomentar una cultura de transparencia.

    1.m)Gestión de Conflictos:

    ● Riesgo: Conflictos dentro del equipo pueden afectar negativamente la moral y la productividad.

    ● Estrategia de Mitigación: Implementar procesos para resolver conflictos de manera efectiva, promover la comunicación abierta y fomentar un ambiente de trabajo colaborativo.

    1.n)Estimación de Tiempos y Recursos:

    ● Riesgo: Subestimar la complejidad de las tareas o la cantidad de recursos necesarios.

    ● Estrategia de Mitigación: Utilizar técnicas de estimación robustas, revisar regularmente el progreso y ajustar las estimaciones según sea necesario.
